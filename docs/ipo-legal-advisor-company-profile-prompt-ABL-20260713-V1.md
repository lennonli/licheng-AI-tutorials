# 提示词：竞聘法律顾问·公司基本情况及法律问题初步分析报告

> 适用场景：竞聘拟上市/拟挂牌企业的法律顾问，需基于公开信息快速产出一份能体现专业能力的公司基本情况报告（Word），用于现场见面或竞标。
> 复用方法：将【目标公司】、【已知背景】、【资料位置】三处占位符替换为实际信息后，整段发送给 AI 助手。

---

## 一、任务目标

1. 阅读我提供的企查查资料包（信用报告、股权穿透报告、关联方认定报告、招投标报告、董监高投资任职及风险报告等）。
2. 结合搜索引擎、官方网站、交易所信息披露、企查查/天眼查等公开渠道，交叉核验并补充公司公开信息（重点：资本市场历史、主营业务、行业地位、上市/挂牌/辅导进展）。
3. 起草一份完整的《公司基本情况及法律问题初步分析报告》（docx 格式），重点梳理**法律层面的初步发现与风险问题**，让公司看到律师的专业能力。
4. 文档生成后直接打开供我审查。

## 三、工作流程（AI 应严格按此顺序执行）

### 步骤 1：阅读内部资料
- 用脚本读取企查查资料包内的：
  - 企业信用报告专业版（docx）——逐段提取正文 + **逐张表格提取**（工商信息、股东、变更记录、对外投资、司法案件、裁判文书、动产抵押、行政许可、资质证书、纳税信用、知识产权、知产出质、企业公告等）。
  - 股权穿透分析报告（xlsx）——提取营业执照信息、公司股东、一级股东穿透。
  - 关联方认定报告（xls）——按关联方维度统计与具体关联方清单。
  - 招投标信用报告（docx）。
  - 董监高投资任职及风险报告（每人均一份 docx）——提取关键人员的对外投资、任职、间接持股、合作企业。
- 注意：信用报告含数十张表格，**表格数据是核心**，不能只读正文段落。

### 步骤 2：检索公开信息（与步骤1并行）
- 至少检索以下维度：
  - 公司新三板挂牌/摘牌历史（证券代码、挂牌日、摘牌日、摘牌原因、异议股东回购承诺）。
  - IPO/上市辅导进展（辅导机构、辅导期数、备案日期、是否变更过板块）。
  - 主营业务、产品、终端客户、行业地位、竞品。
  - 历史股东退出、机构投资者变动、媒体舆情。
- 数据来源优先级：全国股转系统官网 neeq.com.cn ＞ 交易所信息披露 ＞ 巨潮/东方财富 ＞ 官方媒体/红圈所专业文章 ＞ 企查查/天眼查。
- 所有信息必须标注来源；无法核实的标注【待核验】，**严禁虚构条文、案号、数据**。

### 步骤 3：梳理法律问题
- 至少覆盖以下模块，逐项给出"初步发现/风险等级/初步建议"：
  - 历史沿革（含整体变更、增减资、减资程序合规性）。
  - 股权结构与实际控制人（控制权稳定性、一致行动、员工持股平台、股份支付）。
  - 公司治理（董事会/监事会/独董/董秘、独立性）。
  - 资本市场历史（前次挂牌/摘牌、辅导进展、本次路径规划）。
  - 控股参股企业（含境外子公司的 ODI/外汇/税务）。
  - 诉讼仲裁（逐案列示：案号、案由、身份、金额、进展）。
  - 资产抵押与知识产权质押（重点：核心资产质押的融资结构与解质押安排）。
  - 知识产权（商标驳回、专利布局、出质）。
  - 劳动工伤、税务合规、环保合规。
  - 关联方与同业竞争（特别注意"疑似关系企业"，如共用邮箱/电话）。
- 最后形成一张"重点法律问题分级表"（高/中/低），并点名 2-3 项需优先核查的事项——这是展示专业能力的核心抓手。

### 步骤 4：生成 Word 文档
- 用 python-docx（或 docx-js）按律所规范格式生成，**格式规范见第六节**。
- 文档结构（七部分）：封面 → 目录 → 前言 → 一、公司基本情况 → 二、主营业务与经营资质 → 三、资本市场路径回顾 → 四、法律风险事项初步梳理 → 五、重点法律问题初步发现（专项分析） → 六、初步建议与下一步工作 → 七、声明与限制 → 落款页（签署页单独成页）。
- 文件命名：`{公司全称}-公司基本情况及法律问题初步分析报告-ABL-{YYYYMMDD}-V1.docx`，同对话多版本依次 V2、V3。
- 生成后用 `open` 命令直接打开。

## 四、关键质量要点（展示专业能力的地方）

1. **资本市场历史必须查清**：是否曾挂牌、摘牌原因（主动/被动）、摘牌时对异议股东的回购承诺履行情况——这是新三板二次挂牌和北交所上市的必问点，提前指出最显专业。
2. **专利/知产质押要重点提示**：很多制造业拟上市企业存在核心专利大面积质押融资（甚至通过信托通道），关系核心资产权属，是审核高频关注点。
3. **减资程序合规性**：报告期内若有减资，必须核查债权人通知与公告程序。
4. **疑似关联方**：企查查"疑似关系"中，与公司共用邮箱/电话的企业往往是被忽视的关联方或同业竞争主体，必须点名。
5. **不夸大、不虚构**：基于公开信息所作判断，明确标注"初步"，最终结论以正式尽调为准；所有数据标注来源与截止日期。

## 五、常见坑与避坑指南（踩过的坑）

| 坑 | 表现 | 避坑做法 |
|---|---|---|
| 只读信用报告正文段落 | 漏掉司法案件、股东、变更记录等核心数据（这些都在表格里） | **必须用脚本逐张提取表格**，不能只 `doc.paragraphs` |
| 表格列宽超过正文宽度 | 表格溢出页面、错位、无法居中，视觉"很难看" | 列宽总和必须 ≤ 正文宽度（A4 + 左右3.18cm 边距 = **14.64cm**）；每张表列宽之和控制在 14.0—14.4cm |
| python-docx 设了列宽却不生效 | Word/WPS 打开后列宽被自动重排 | 同时设置三处：`w:tblLayout type=fixed` + `w:tblW` 总宽度 + `w:tblGrid/gridCol` 列宽 |
| 表格对齐属性用错 | `WD_TABLE_ALIGNMENT.JUSTIFY` 不存在导致报错 | 用 `WD_TABLE_ALIGNMENT.CENTER`，并通过 `w:jc` 控制 |
| 版本管理混乱 | 修改后覆盖原文件、找不到历史版本 | 同对话内多版本严格按 V1/V2/V3 递增，不覆盖 |
| 生成后不自动打开 | 律师看不到成果 | 脚本最后一步用 `open` 命令打开文件 |

## 六、Word 文档格式规范（律所默认）

### 正文与标题
- 首页主标题：黑体，小三号（15pt），加粗，居中，首行缩进0，1.5倍行距，段前1.5行。
- 一级章标题：黑体，小三号（15pt），不加粗，居中，首行缩进0，段前段后各1行。
- 二级标题（"（一）"）：宋体，四号（14pt），加粗，两端对齐，首行缩进0，段前段后各0.5行。
- 三级/四级标题：宋体，小四号（12pt），加粗，首行缩进2字符。
- 普通正文：宋体，小四号（12pt），首行缩进2字符，段前段后各0.25行，1.5倍行距。
- 数字使用 Times New Roman；1,000 以上数字用千位分隔符。

### 表格
- 外边框 1.5 磅，内边框 0.5 磅；表头行底色浅灰（如 DDDDDD）、居中加粗。
- 数据行：文字左对齐，数字右对齐，序号/短字段居中；单元格内容垂直居中。
- 列宽分配原则：**序号列最窄（0.8—1.2cm）、长文本列最宽、短字段列窄**；列宽总和 ≤ 14.5cm。
- 表格整体居中（`table.alignment = CENTER` + `w:jc val=center`）。
- 表格字体：宋体五号（10.5pt），内容密集时用小五（9.5—10pt）；表头加粗。
- 表格下方注释：宋体五号，首行缩进2字符，单倍行距，段前段后0。

### 其他
- A4 纸，上下边距 2.54cm，左右边距 3.18cm。
- 页脚居中页码，宋体五号（用 `PAGE` 域）。
- 涉及签署/签字页的，签署页单独成页，用分页符分隔。
- 落款：上海市锦天城（深圳）律师事务所，经办律师李成，注明出具日期。

## 八、文件命名与版本管理

- 命名规则：`{简称或全称}-{文件类型}-ABL-{YYYYMMDD}-V{版本号}`
- 同一对话中产生多个版本，依次 V1、V2、V3 递增，**不覆盖**已有版本。
- 律所固定标识：ABL；经办律师：李成；律所全称：上海市锦天城（深圳）律师事务所。
